Taking Input from user:

By now, you have learned how to store values and process them. All this may come to use only when you know how to take user input. The user input is actually mostly done with the HTML, but since you haven't reached that level yet, a more basic and easier way of taking it is the prompt function.

The prompt function provides an input field to enter text. It needs one value, which is the text that should appear above the text input field. Here's how it runs:


						var inputFromUser = prompt("What is your name?");

This will show an alert box like this:

And anything entered in the text field will be stored in the variable inputFromUser.

A bit detail about it:

With the above information, you should be able to get input from the user just fine. But here are a few things to to remember:

  • When this popup appears, the whole webpage gets paused. So if you are trying to input JavaScript in the console while this prompt is present, the console will refuse to respond. The same scenario is observed with alert
  • When the OK button is pressed without filling the input field, or the Cancel button is pressed, undefined is given as user input. To prevent undefined from interfering with our work, we will need to check it using if statements, which you will learn below.
  • The prompt functions only returns a string, even when a number is inputted. So, for this purpose, you will need to convert the number into a string form before performing arithmatic on it.

Type Conversion:

By now, you know that there are strings and numbers in JavaScript:


						var items = "5"; // This is a string, different from
						var items = 5;   // This, which is a number

Meaning that a number functions differently with operators. This is mostly fine, but the problem appears when you have to add a string, which contains a number digit, such that:


						var firstNum = "5";
						var secondNum = "10";

						console.log(firstNum + secondNum); // Prints "510" instead of 15

Trying to add these together will simply make then join with each other, because they are strings, and strings are concatenated when the plus operator is used. So is it programatically impossible to add these numbers inside the strings?

No obviously, but the point is to tell the solution. It's rather easy, and is really useful when working with these sort of values:


						parseInt(string)

Where string is the string that needs to be

The parseInt() function takes in the string to convert as one argument (input), and it gives out (returns) the converted value. Remember that any returned value can be supplied to anything, whether it's a console.log, alert, or just storing the value in a variable.


						var myFavouriteNumber = "7"; 
						// How do we convert it to a number?
						var myFavouriteActuallyNumber = parseInt(myFavouriteNumber);

						// Now myFavouriteActuallyNumber contains the number 7
						// We can print out the converted number without storing it too:
						console.log("My Favourite number in a number form:", myFavouriteActuallyNumber);

Using commas in console.log():

As observed above, I used commas , to separate the two values. This doesn't act as a concatenation operator at all, but it's just a functionality of the console.log() function that it can accept multiple arguments and print the out separately in the same line.

The comma , causes each of the values to print out separately, and not get concatenated with each other. This way, you will be able to see the myFavouriteActuallyNumber value show up in purple, indicating that its a number. If the plus + operator was used instead, the value would have merged (concatenated) into the string before it, and the difference would not be signified.

If statements:

Now, by now, all you know about is to print, input and store values. But sometimes you need to have a decision in your program. You need to use logic, to specify certain instructions. Remember logic gates? Well, if statements are a logical application of it in programming.

They exist in every programming language in some sort of way (not HTML, HTML is just a joke). They are abundently used in programming, and are really simple and helpful. In JavaScript, they are represented like this:


						if (condition) {
							// if body (the condition was matched)
						}

						// Or a more compact form:
						if (condition) /* if body */

Where condition can be any expression returning a value.


						var isDocumentEasy = true;
						if (isDocumentEasy) { // This condition should match, since the isDocumentEasy variable is true
							console.log("This document is easy :)")
						}

Symbolically speaking, the code inside the if statement is only executed, if the condition evaluates to true. The reason I used the word "evaluates" is because the condition doesn't need to be exactly a true boolean value, but anything that is taken as a true in boolean programming. Here's a generic list of values that should evaluate to specific boolean values:

  • Most regular variables evaluate to true. All boolean values obviously evaluate to their own value.
  • A number 0 evaluates to false. This can be overridden by using strict comparisons, which are discussed down below.
  • Empty strings (i.e. strings with nothing in them, like "") evaluate to false.
  • All special entities like undefined, null, NaN (excluding Infinity) evaluate to false.

						var aZeroNumber = 0;
						var anEmptyString = "";
						var aBooleanValue = true;
						
						if (aZeroNumber) console.log("Condition Matched"); // This will not run.
						if (anEmptyString) console.log("Condition Matched"); // This will also not run.
						if (aBooleanValue) console.log("Condition Matched"); // This will run.

Condition Operators:

The condition being supplied to the if statements is not limited to values, but a large range of expressions. Here's how to tackle with various situations while working with if statements:

  • Matching multiple conditions:

    To compare multiple values in if statements, you can use the && or || operators. They are used between two conditions, and decide according to their function.

    The && operator is used to match both of the conditions (between which the operator is present). If any of the target conditions evaluates to false, it returns false. This operator behaves the same as the AND logic gate.

Command Line Introduction

Introduction and common points to know about the Windows/Linux/Mac command line.

Introduction:

Up until now, you probably have only used cmd for cool quirky stuff. Maybe copy pasting some lines from the internet hoping those fix your Rainbow Six Siege not connecting to the internet.

Learning command line basics is something you should already know before programming. Command line isn't difficult, but it can get as complex as you want it to be. However, some vital basics are explained here.

Platforms:

Different platforms have different commmand lines. Although they behave similar to each other, they do have some major differences. Windows is far apart from Linux and Mac, whereas the latter two are similar.

Directories:

Directories are the "folders" on the storage. Each directory (folder) can have a file or another directory, and so on.

Current Directory:

As you may have noticed, all of the terminals are showing a path. In Windows, the path C:\Users\Thomas is apparent. Linux and Mac show the / which means the root (topmost) directory.

This specified directory is the current directory the terminal session has. Applications can use the current directory for their work. For example, if you are about to create a new Node project, running the npm init command will create a project in that directory

Home directory

As observed above, the Mac terminal shows ~ as the path. The ~ directory is called the Home directory. It is used in Mac and Linux.

Changing the current directory:

Use the cd command to change the directory to the specified one:


            cd /Users

Path Separators:

In windows, to separate two directories in a path, the backslash character \ is used, whereas in linux and mac, the forward slash / is used. Windows now supports both slashes, but I've seen that \ works better with autocompletion.

Relative Path:

Sometimes you want to specify path relative to the current path. For example, if you want go to the previous directory (parent directory). The . and .. are used to specify relative paths.

Path Meaning
./ Explcitly specify current directory. E.g, if you want to refer file.txt in the current directory, you can do ./file.txt
../ Previous directory. This refers to the directory parent to the current directory. E.g., if you are in C:\Users\Thomas\ then doing a cd ../ will change the directory to C:\Users\

Let's finish this off with an example. Suppose we are in /Foo and want to go to /Bar, then you should do this:


            cd ..\Bar

and then the path will be changed:

Shortcuts:

Almost all command lines provide a set of shortcuts to effeciently run commands. Command lines are different from your regular text fields, so copy and paste usually doesn't work as-is. Here's a list of important shortcuts:

Shortcut Description
↑/↓ Cycle through the history of all the commands that you have attempted to execute. Very handy when you made a slight mistake in the last command and you want to correct it.
↹ Attempts to autocomplete the word you're typing to a path or a program, etc.
Ctrl + C or Ctrl + X Exit the currently running task.

Copying and pasting:

In Windows, click and drag through the text you want to copy, then right click on it. The selected text will be copied once the right click is pressed. Similarly, to paste some text, right click anywhere without selection.

Linux and Mac usually provide a simple context menu option to copy or paste text.

Executing programs:

Generally, anything written in the command line is written in this form:


            program-name param1 param2 param3 ...

Where param1, param2 and param3 are the values given to the program.

So, when we are doing npm init, we are running the npm program, and giving the value init to it.

We have to always make sure that the computer understands what program-name refers to.

NodeJS Introduction

Here's a little NodeJS introduction document that might help you understand what's NodeJS.

Introduction:

NodeJS is just server-sided JavaScript. It takes quite the effort to explain that phrase but if you understand it now, then that's pretty much what NodeJS is. If you google NodeJS you get:

Node.js is a back-end JavaScript runtime environment. Node.js runs on the V8 JavaScript Engine and executes JavaScript code outside a web browser.

The JavaScript you know:

You make an html file. You have some HTML in it. Maybe links to your beautiful CSS, and JavaScript. When you open that file, that file is given to the chrome browser (or any other shitty browser you may use). Chrome looks at the code, all the HTML, CSS and JavaScript. And executes it right away.

This is a simple model without any servers or anything. Code, given to a software. That's it. The JavaScript written there is client-sided, meaning it executes on the client's computer.

Server-sided JavaScript:

Now, if you go ahead and open a chrome tab, and go to google.com and press enter, it shows a spinny. Then after some time the spinny turns blue, means it got a response. When the response is received, it's executed normally like the JS you are aware of.

Node is just one technology to program servers. google.com probably runs on a hundred times more advanced software. Let's take an imaginary website called Gaagle.com which simply uses NodeJS for backend.

When you go to gaagle.com, the browser sends an HTTP Request to the gaagle.com server. The server isn't a magical black box. It can be a simple computer accepting HTTP requests, and simply responding them back. So, gaagle.com also has some JavaScript code, written with NodeJS, which accepts HTTP Requests, and responds to them.

That code is on the server side. It resides on the server, answers any requests, and responds with some HTML, for your browser to run.


That JavaScript is called Server-sided JavaScript. Node is used to program servers. It runs on the server side, is installed on the server.

NodeJS isn't exactly HTTP Web Servers

If you want to create an application exactly like gaagle.com, then you'd need Express.js, a NodeJS library. Most of the time Node is used as a boilerplate for the popular libraries like ReactJS, Express.js, and many more.

Installing NodeJS:

Head over to their download page, download setup, and install. While installing, make sure any option related to "Adding to PATH" is checked (if you don't find one, then it's okay).

Once installed, open up Command prompt (or terminal), and enter:


            node -v

You should see the version of NodeJS installed. If so, then you're good to go.

Prerequisites:

Before NodeJS, there's some things you should know about to properly learn NodeJS. Good thing is that I'd be covering most of it in this document:

  • JavaScript: I mean of course, NodeJS literally has JS in the name. You must know some JavaScript before getting started with Node.
  • Packages and NPM: NPM is Node Package Manager. As I said, NodeJS acts as a boilerplate for most of the popular libraries. These libraries install as a package, through a package manager. You'll learn more about these below. It's a part of Node, but people don't really care to explain it well.
  • Basic command line knowledge: You will be using NodeJS and NPM with cmd or terminal, depending on your platform. cmd from Windows works a bit differently than a linux terminal. I'll cover cmd because that's what most of the people use.

Packages and NPM:

Node is mostly all about it's packages. A package is a library. A library is some helping code files for a project. C language has native libraries (stdlib.h, stdio.h, etc.). C++ also has some native libraries (iostream).

A package is structurally a Node project, being used in another Node project. NPM stands for Node Package Manager. It acts like the Play store for packages. You can browse, search and install packages from NPM.

If you go to any Node library, they will always have an Installation section guiding to install their package with it's name.

A bit deeper into NPM:

NPM comes with Node. It's a package manager, like any other package manager. NodeJS projects are managed with npm as well. To create a project, install it's dependencies, and more. node is only used to execute a NodeJS project.

NPM is also used to create a NodeJS project. You can execute npm init -y to create a package.json for a new project. This file will be explained later on.
